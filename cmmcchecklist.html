AC.L2-3.1.1 - Limit System Access to Authorized Users

✓ REMEDIATED

🎯 What This Means in Plain English

Only people who are supposed to use your computers and systems should be able to log in. Everyone needs their own username and password. No sharing accounts!

⏱️ Difficulty: Moderate | Time: 2-4 hours
⏰ Time Needed: 2-4 hours
👤 Who Should Do This: IT Admin or Domain Admin
💰 Cost: $0 (built into Windows)
1 Review All User Accounts in Active Directory
📍 What We're Doing

We're going to look at all the user accounts in your system to make sure everyone who has an account should have one.

👆 Click-by-Click Instructions

  1. Log into your Domain Controller server
  2. Click the Start button (Windows icon)
  3. Type dsa.msc and press Enter
  4. This opens "Active Directory Users and Computers"
  5. In the left panel, click the arrow next to your domain name to expand it
  6. Click on the "Users" folder
  7. You'll see a list of all user accounts
Start → Type "dsa.msc" → Enter → Expand Domain → Click "Users"

💡 What to Look For

  • Accounts for people who no longer work here (these should be disabled)
  • Generic accounts like "Admin" or "Test" that multiple people might use
  • Accounts you don't recognize
2 Export a List of All Users (For Documentation)
📍 What We're Doing

We're going to create a spreadsheet of all users so you can review it and keep it as evidence for your CMMC assessment.

👆 Click-by-Click Instructions

  1. Open PowerShell as Administrator (see Prerequisites above)
  2. Copy the command below by highlighting it and pressing Ctrl+C
  3. Click in the PowerShell window and press Ctrl+V to paste
  4. Press Enter to run the command
Get-ADUser -Filter * -Properties Name, SamAccountName, Enabled, LastLogonDate, Description, Department | Select-Object Name, SamAccountName, Enabled, LastLogonDate, Description, Department | Export-CSV -Path "C:\CMMC_Evidence\User_Account_List.csv" -NoTypeInformation

⚠️ If You Get an Error

If you see a red error message about the folder not existing, run this command first:

New-Item -Path "C:\CMMC_Evidence" -ItemType Directory -Force

💡 Where to Find Your File

Open File Explorer, go to C:\CMMC_Evidence, and you'll see a file called User_Account_List.csv. You can open this in Excel.

3 Disable Accounts for People Who Left the Company
📍 What We're Doing

When someone leaves the company, we disable their account so they can't log in anymore.

👆 Click-by-Click Instructions (Using the GUI)

  1. Open Active Directory Users and Computers (Start → dsa.msc)
  2. Navigate to the Users folder
  3. Find the account you want to disable
  4. Right-click on the account name
  5. Click "Disable Account"
  6. You'll see a red X appear on the account icon - this means it's disabled
Right-click user → Disable Account → OK
💻 Alternative: Using PowerShell (Faster for Multiple Accounts)

Replace "john.smith" with the actual username:

Disable-ADAccount -Identity "john.smith"

🚨 Important

Don't delete accounts - disable them instead. You may need them for audit trails or legal reasons.

IA.L2-3.5.7 - Enforce Minimum Password Complexity

⏳ NOT DONE

🎯 What This Means in Plain English

Passwords must be strong enough that they can't be easily guessed. This means requiring long passwords with a mix of letters, numbers, and special characters.

⏱️ Difficulty: Easy | Time: 20-30 minutes
1 Open Group Policy Management and Navigate to Password Policy

👆 Click-by-Click Instructions

  1. On your Domain Controller, press Start
  2. Type gpmc.msc and press Enter
  3. Expand Forest → Domains → [Your Domain]
  4. Right-click "Default Domain Policy"
  5. Click "Edit..."
  6. Navigate to: Computer Configuration → Policies → Windows Settings → Security Settings → Account Policies → Password Policy
Computer Configuration → Policies → Windows Settings → Security Settings → Account Policies → Password Policy
2 Set Minimum Password Length

👆 Click-by-Click Instructions

  1. Double-click "Minimum password length"
  2. Check "Define this policy setting"
  3. Set the value to 14 characters
  4. Click OK
Minimum password length: 14 characters

💡 Why 14 Characters?

NIST now recommends longer passwords over complex ones. A 14-character password is much harder to crack than an 8-character password, even with special characters.

3 Enable Password Complexity Requirements

👆 Click-by-Click Instructions

  1. Double-click "Password must meet complexity requirements"
  2. Select "Enabled"
  3. Click OK

💡 What Complexity Means

When enabled, passwords must contain at least 3 of these 4 categories:

  • Uppercase letters (A-Z)
  • Lowercase letters (a-z)
  • Numbers (0-9)
  • Special characters (!@#$%^&*)

Also, the password cannot contain the username or parts of the full name.

4 Set Password History (Prevent Reusing Old Passwords)

👆 Click-by-Click Instructions

  1. Double-click "Enforce password history"
  2. Set to 24 passwords remembered
  3. Click OK
Remember 24 passwords
5 Set Maximum Password Age

👆 Click-by-Click Instructions

  1. Double-click "Maximum password age"
  2. Set to 365 days (or 0 for no expiration if using MFA)
  3. Click OK

💡 Modern Guidance

NIST 800-63B now recommends NOT requiring regular password changes unless there's evidence of compromise. If you use MFA (highly recommended), you can set this to 0 (no expiration) or a longer period like 365 days.

6 Set Minimum Password Age

👆 Click-by-Click Instructions

  1. Double-click "Minimum password age"
  2. Set to 1 day
  3. Click OK
Minimum password age: 1 day

💡 Why This Matters

This prevents users from quickly changing their password 24 times to get back to their old password.

7 Apply and Verify
gpupdate /force

Then verify the settings:

net accounts

💡 Expected Output

You should see:

  • Minimum password length: 14
  • Password history length: 24
  • Maximum password age (days): 365 (or Unlimited)
  • Minimum password age (days): 1

  • Screenshot of Password Policy settings in Group Policy
  • Screenshot of "net accounts" output

3 Increase Security Event Log Size
📍 What We're Doing

By default, Windows only keeps a small amount of logs. We need to increase this so logs aren't deleted too quickly.

👆 Click-by-Click Instructions (GUI Method)

  1. Press Windows Key + R on any server or computer
  2. Type eventvwr.msc and press Enter
  3. In the left panel, expand "Windows Logs"
  4. Right-click on "Security"
  5. Click "Properties"
  6. Change "Maximum log size (KB)" from the current value to 4194240 (4GB)
  7. Select "Overwrite events as needed"
  8. Click OK
Win+R → eventvwr.msc → Windows Logs → Right-click Security → Properties

Or use this PowerShell command (faster):

# Set Security Event Log to 4GB maximum size wevtutil sl Security /ms:4194304000 # Set Application Event Log to 500MB wevtutil sl Application /ms:524288000 # Set System Event Log to 500MB wevtutil sl System /ms:524288000 Write-Host "Event log sizes updated successfully!" -ForegroundColor Green # Verify the settings Write-Host "`nCurrent Security Log Size:" -ForegroundColor Yellow wevtutil gl Security | Select-String "maxSize"
4 Set Up Automatic Log Archiving (Keep Logs for 1 Year)
